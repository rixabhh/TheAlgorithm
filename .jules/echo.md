## 2026-04-03 — Onboarding & Form UX Polish
**UX Issue:** Uploading a chat had zero immediate feedback. Users were left wondering if the file was recognized, and hitting "Decode" just showed a frozen button while processing. The dashboard also featured text walls, making it hard to scan insights. Finally, the app lacked strong trust signals right where users input their API key.
**Root Cause:** The UI elements were either minimal or lacking intermediate states. Feedback required full progression to the next screen, and inputs had insufficient labels.
**Solution:** I implemented a `detectionCard` that reads the first 1MB of the file `onChange` and shows the detected platform + message count. I also added a step-based progress interval so the user sees ("Parsing...", "Redacting...", "Generating..."). Finally, I tightened the reading width (`max-w-prose`) in the AI Insights dashboard layout. ARIA labels were added, along with `transition: all 0.3s ease-in-out` on buttons.
**Pattern:** Always provide immediate, client-side visual feedback for asynchronous, potentially long-running user actions. Constrain long-form content reading width to ~65 characters (`max-w-prose`).
