## 2024-04-16 — Security & Privacy Hardening
**Vulnerability/Gap:** Missing HTTP security headers, missing rate limiting on local endpoints, generic error traces leaking out, LLM fetch lacking timeouts leading to hanging connections, API keys visible in global catch blocks, JS `console.log` leaking state, and chat arrays lingering in garbage collector.
**Root Cause:** Rapid MVP development lacking defense-in-depth and strict boundaries on memory management.
**Fix Applied:** Added CSP/headers to `app.py`, created an in-memory rate limiter `is_rate_limited`, added upload validation `validate_upload`, set `AbortController` timeouts for `fetch` in API functions, masked API keys via regex replacement in catch blocks, removed console logs from the frontend, and explicitly set chat array lengths to 0 in a `finally` block before setting to null.
**Remaining Risk:** Rate limiting is in-memory and resets on server restart. File upload validation is a helper function but isn't explicitly hooked up to an actual upload endpoint in `app.py` since the current flow is strictly client-side parsing; however, the validation mechanism is available if server-side processing is implemented.
